- identity-hub-frontend만 9개 항목 있음
- 나머지 프로젝트: backlog.md 없거나 비어있음
- **에이전트가 `@dev backlog` 해도 할 일이 없음**

#### 2. 일부 프로젝트 테스트 인프라 부재
| 프로젝트 | 테스트 | 비고 |
|----------|--------|------|

- codex:rescue 에스컬레이션도 트리거 불가

#### 3. Gemini/Codex Phase 0가 FastAPI 프로젝트에만 적용
- CI3/PHP, Kotlin, Docker 프로젝트는 Phase 0 스킵
- dev.md에 `no Gemini/Codex` 명시된 프로젝트 있음 (speakingmax-backend, maxai-b2c-backend)

#### 4. 자율 실행 오케스트레이터 부재
- `@dev backlog 전체`를 **누군가** 트리거해야 함
- 프로젝트 간 순차/병렬 오케스트레이션 없음
- 크론 기반 자동 실행 미설정

## 결론: 자율 업무 수행 가능한가?

**설정 인프라는 90% 완비**. 부족한 건 "할 일(백로그)"과 "자동 트리거".

```
현재: 사람이 "@dev 이거 해" → 에이전트 자율 라우팅/실행/리뷰/테스트 ✅
목표: 에이전트가 스스로 백로그 확인 → 우선순위 판단 → 실행 → 완료 ❌
```

### 실제로 필요한 것

| 필요 항목 | 난이도 | 설명 |
|-----------|--------|------|
| **백로그 채우기** | 낮음 | 각 프로젝트 TODO/이슈를 backlog.md로 수집 |
| **크론 트리거** | 중간 | `/start` 스킬을 크론으로 자동 실행하거나, 세션 시작 시 `@dev backlog` 자동 호출 |
| **PHP 프로젝트 테스트 추가** | 높음 | PHPUnit 설정 + 기본 테스트 작성 |
| **오케스트레이터 스킬** | 중간 | 멀티 프로젝트 순회하며 `@dev backlog` 실행하는 스킬 |

## 실행 계획

### Step 1: 백로그 수집 (각 프로젝트)
- GitHub Issues, Linear 티켓, 코드 TODO, 기존 문서에서 미완료 항목 수집
- `docs/backlog.md`에 우선순위별 정리
- 대상: backlog 비어있는 프로젝트 전부

### Step 2: 자동 트리거 메커니즘
- 옵션 A: `/start` 스킬에 멀티 프로젝트 순회 로직 추가
- 옵션 B: 크론 스킬로 주기적 `@dev backlog` 트리거
- 옵션 C: 세션 시작 훅에서 active/ 미완료 항목 자동 표시

### Step 3: 테스트 인프라 보강 (선택)
- PHP 프로젝트에 PHPUnit 기본 설정
- 최소한 smoke test 수준

### Step 4: 멀티 프로젝트 오케스트레이터
- 프로젝트 간 의존성 고려한 실행 순서 결정
- 병렬 가능한 프로젝트는 동시 실행

## 검증 방법
1. 각 프로젝트에서 `@dev backlog` 실행 → 태스크 픽업 확인
2. `@dev active` 실행 → 파이프라인 전체 흐름 확인
3. 크로스 프로젝트: `@team` 스폰 → 연관 프로젝트 에이전트 협업 확인

## 수정 대상 파일
- 각 프로젝트 `docs/backlog.md` (생성 또는 업데이트)
- `~/.claude/skills/` (오케스트레이터 스킬 신규)
- 각 프로젝트 `docs/active/` (태스크 파일)
