🔴 high priority (즉시 액션):

      
    
        
  • too_short: 출력 500자 미만 — agent 가 작업 거부했거나 결과 누락. 액션: kickoff prompt 더 명확히 작성. "다음 5단계로 답해" 같은 구조 강제.
    • 
        
  • domain_zero: 자기 knowledge 용어 0개 매칭 — 빌드 미반영 or knowledge 부적합. 액션: ~/.claude/agents/build-agents.sh {agent} 재빌드, 또는 knowledge .md 점검.
    • 
        
  • wrong_agent: 다른 agent 가 더 적합. 액션: 다음번엔 추천 agent 호출. 같은 패턴 반복되면 라우팅 룰 추가.
    • 
      

      
🟡 medium priority (개선 검토):

      
    
        
  • low_decision: 의사결정 패턴 거의 없음 (근거/대안/검증 합 < 2). 액션: agent system prompt 에 "결정 시 근거+대안 명시" 추가.
    • 
        
  • no_reasoning: 근거/트레이드오프 0건. 액션: "Why this choice over X?" 질문 강제.
    • 
        
  • shallow_domain: 긴 출력인데 도메인 용어 1~2개만. 액션: kickoff 에 자기 knowledge 도메인 명시 (예: "data-analyst knowledge 의 SQL 최적화 패턴 적용").
    • 
      

      
⚪ low priority (선택적):

      
    
        
  • no_structure: 헤더/리스트/표/코드 합 < 3 — 산문형 출력. 액션: 출력 템플릿 (## 분석, ## 권고, ## 검증) 적용.
