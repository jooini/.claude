## 1. 버그 라이프사이클

```
발견 → 등록 → 분류/우선순위 → 할당 → 수정 중 → 수정 완료
                                              → 검증 → 닫힘
                                              → 재오픈 (재현됨)
```

---

## 2. 버그 리포트 작성

```markdown
## 제목
[모듈] 증상 요약 — 간결하고 명확하게

예: [결제] iOS Safari에서 카드 등록 후 즉시 삭제 시 앱 크래시

## 기본 정보
- 심각도: Critical / High / Medium / Low
- 우선순위: P1 / P2 / P3 / P4
- 환경: iOS 17.2, Safari, iPhone 14 Pro
- 발견일: 2024-01-15
- 발견자: QA팀 김QA
- 담당 개발자: 홍길동

## 재현 단계
1. 로그인 후 마이페이지 > 결제수단 접근
2. "+ 카드 추가" 클릭
3. 유효한 카드 정보 입력 후 "등록" 클릭
4. 등록 완료 즉시 "삭제" 버튼 클릭
5. 확인 팝업에서 "삭제" 클릭

## 기대 결과
카드가 삭제되고 빈 목록 표시

## 실제 결과
앱이 강제 종료됨. 재실행 시 카드는 남아있음.

## 재현율
5/5 (100%)

## 첨부 자료
- [영상: crash_recording.mp4]
- [로그: crash_log.txt]

## 추가 정보
- Android Chrome에서는 재현 안 됨
- 카드 등록 후 1초 대기 후 삭제 시 재현 안 됨 (타이밍 이슈 추정)
```

## 3. 심각도 vs 우선순위

```
심각도 (Severity) — 기술적 영향
  Critical: 서비스 불가, 데이터 손실
  High:     주요 기능 동작 불가
  Medium:   기능은 되지만 UX 저하
  Low:      사소한 UI 이슈

우선순위 (Priority) — 비즈니스 관점
  P1: 즉시 수정 (당일)
  P2: 다음 릴리스 전 수정
  P3: 계획된 스프린트 내
  P4: 시간 있을 때

심각도 ≠ 우선순위:
  예1: 홈페이지 로고 오타 → 심각도 Low, 우선순위 P1 (CEO 요청)
  예2: 복잡한 엣지케이스 크래시 → 심각도 High, 우선순위 P3 (발생 빈도 매우 낮음)
```

## 4. 버그 트리아지 (Triage)

```
주기: 매일 또는 격일 (팀 규모에 따라)
참여: QA 리드, 개발 리드, PM

진행:
1. 신규 버그 검토
2. 재현 확인 (담당자 지정)
3. 심각도/우선순위 합의
4. 스프린트 배정 또는 Known Issue 등록
5. 릴리스 차단 여부 결정

릴리스 차단 기준:
  - Critical 버그 있으면 무조건 차단
  - High 버그 2개 이상 → 차단 검토
  - P1 버그 미해결 → 차단
```

## 5. 재현 불가 버그 처리

```
재현 불가 버그 처리 프로세스:

1. 추가 정보 요청
   - OS/브라우저/앱 버전
   - 계정 정보 (테스트 계정)
   - 네트워크 환경
   - 로그/스크린샷

2. 환경 복제 시도
   - 제보자 환경과 동일하게 설정
   - 다양한 디바이스 시도

3. 로그 분석
   - Sentry 에러 로그 확인
   - 서버 액세스 로그 확인

4. 상태 처리
   - 3회 시도 후 재현 불가 → "Cannot Reproduce" 처리
   - 재발 시 다시 열기
```

## 6. 버그 메트릭

```
추적해야 할 지표:

탈출 버그율 (Escaped Defect Rate):
  운영 발견 버그 / 전체 버그 → 낮을수록 좋음

평균 해결 시간 (MTTR):
  P1: 목표 4시간 이내
  P2: 목표 2일 이내

재오픈 비율:
  수정 후 다시 열린 버그 / 닫힌 버그
  → 10% 초과 시 수정 프로세스 검토

스프린트 신규 버그 수:
  증가 추세면 품질 저하 신호
```

---

## 7. 안티패턴

- **재현 단계 없는 버그 등록**: 개발자가 디버깅 불가
- **모든 버그를 P1으로**: 실제 중요한 것을 구분 못함
- **버그 방치**: 미해결 버그가 100개+ 쌓이면 관리 불가
- **수정 확인 없이 Close**: QA 검증 후 Close
- **버그 원인 분석 없음**: 재발 방지를 위한 근본 원인 파악 필요
