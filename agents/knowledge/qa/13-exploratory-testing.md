## 1. 탐색적 테스트란

사전 스크립트 없이 테스터의 지식과 직관으로 탐험하며 버그 발견.
계획된 테스트가 놓친 예상치 못한 버그를 잡는 데 효과적.

**언제 사용:**
- 신규 기능 첫 번째 검토
- 계획 테스트 후 추가 탐색
- 복잡한 비즈니스 로직
- 릴리스 전 최종 검증

## 2. 세션 기반 탐색적 테스트 (SBET)

```markdown
# 탐색적 테스트 세션

세션 차터 (Charter):
  목적: 결제 플로우의 엣지 케이스 탐색
  범위: 장바구니 → 결제 완료
  시간: 60분

환경:
  URL: https://staging.example.com
  계정: qa-test@example.com
  브라우저: Chrome 최신

탐색 아이디어:
  □ 결제 도중 뒤로가기 버튼
  □ 같은 상품을 여러 탭에서 동시 구매
  □ 재고 1개 → 수량 2개 입력
  □ 쿠폰 코드 + 포인트 동시 사용
  □ 이미 사용한 쿠폰 재사용
  □ 네트워크 끊김 후 재연결 시 주문 상태
  □ 결제 완료 페이지에서 새로고침
  □ 뒤로가기로 결제 페이지 재접근 후 재제출

발견 사항:
  [시간] [현상] [재현 방법] [심각도]
  14:05  장바구니 수량 -1 입력 가능  수량 입력에 -1 입력  Medium
  14:22  결제 완료 후 새로고침 시 중복 주문 발생  ...  Critical

소요 시간: 55분
버그 수: 3개 (Critical 1, Medium 1, Low 1)
```

## 3. 탐색 휴리스틱 (SFDIPOT)

```
Structure    — 구조적 탐색
  무엇으로 만들어졌는가? 컴포넌트 분해, 의존성

Function     — 기능 탐색
  무엇을 하는가? 각 기능의 입출력

Data         — 데이터 탐색
  어떤 데이터를 처리하는가? 경계값, 특수문자, null

Interface    — 인터페이스 탐색
  어떻게 상호작용하는가? 버튼, 폼, API

Platform     — 플랫폼 탐색
  어디서 실행되는가? OS, 브라우저, 디바이스

Operations   — 운영 탐색
  실제로 어떻게 쓰이는가? 사용자 시나리오

Time         — 시간 탐색
  시간에 따라 어떻게 변하는가? 만료, 지연, 순서
```

## 4. 마인드맵 기반 탐색

```
로그인 기능
├── 정상 케이스
│   ├── 이메일 + 비밀번호 로그인
│   ├── 소셜 로그인 (구글, 카카오)
│   └── "로그인 유지" 옵션
├── 에러 케이스
│   ├── 존재하지 않는 이메일
│   ├── 잘못된 비밀번호
│   ├── 잠긴 계정
│   └── 네트워크 오류
├── 경계 케이스
│   ├── 이메일 최대 길이
│   ├── 특수문자 포함 비밀번호
│   └── 공백 포함 입력
└── 보안
    ├── 브루트포스 시도
    ├── SQL Injection
    └── 세션 하이재킹 시도
```

## 5. 버그 리포트 작성

```markdown
제목: [결제] 결제 완료 후 새로고침 시 중복 주문 발생

심각도: Critical
우선순위: P1
발견자: QA팀 김QA
발견일: 2024-01-15
환경: Chrome 120, Windows 11, Staging

재현 단계:
1. 로그인 후 상품 1개 장바구니 추가
2. 결제 진행 → 완료
3. 주문 완료 페이지에서 F5 새로고침
4. 브라우저 "양식 다시 제출?" 확인 클릭

기대 결과:
  - 새로고침 방어: "이미 완료된 주문입니다" 메시지 또는 주문 내역 페이지로 리다이렉트

실제 결과:
  - 동일한 주문이 중복 생성됨
  - 결제가 두 번 청구됨

증거:
  - [스크린샷 첨부]
  - [영상 첨부]

비즈니스 임팩트:
  중복 결제로 고객 불만 및 환불 처리 비용 발생

관련 이슈: #1234
```

## 7. 안티패턴

- **기록 없는 탐색**: 발견한 것 즉시 메모 → 나중에 재현 불가
- **시간 제한 없음**: 타임박싱 (60~90분) → 집중도 유지
- **같은 영역만 반복 탐색**: 차터로 범위 설정
- **버그 제보 미루기**: 발견 즉시 등록 → 잊어버림
- **개발자와 소통 없이 탐색**: 복잡한 비즈니스 로직은 함께 이해
