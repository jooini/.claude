## 1. QA 지표가 중요한 이유

측정하지 않으면 개선할 수 없다.
QA 메트릭으로 프로세스 약점 파악 → 집중 개선.

## 2. 핵심 QA 메트릭

### 결함 관련

```
결함 탈출율 (Defect Escape Rate)
  = 운영 발견 버그 수 / 전체 버그 수 × 100
  → 낮을수록 좋음 (5% 이하 목표)

결함 밀도 (Defect Density)
  = 버그 수 / 코드 라인 수 (KLOC)
  → 추세 추적 (증가면 품질 저하 신호)

결함 재오픈율 (Defect Reopen Rate)
  = 재오픈 버그 / 닫힌 버그 × 100
  → 10% 이상이면 수정 프로세스 문제

심각도별 버그 분포
  Critical / High / Medium / Low 비율
  → Critical이 높으면 위험
```

### 테스트 효율성

```
테스트 케이스 실행율
  = 실행된 TC / 전체 TC × 100
  → 90% 이상 목표

테스트 통과율 (Pass Rate)
  = 통과 TC / 실행 TC × 100
  → 95% 이상 (낮으면 품질 또는 TC 문제)

자동화율 (Automation Rate)
  = 자동화된 TC / 전체 TC × 100
  → 반복 TC는 70% 이상 자동화 목표

Flaky Test 비율
  = 불안정한 테스트 / 전체 자동화 테스트
  → 5% 이하 목표
```

### 생산성

```
평균 버그 해결 시간 (MTTR)
  P1: 4시간 이내
  P2: 2일 이내
  P3: 스프린트 내

테스트 실행 시간
  PR CI: 5분 이내
  전체 회귀: 30분 이내
  → 초과 시 최적화 필요

스프린트 버그 밀도 추세
  스프린트별 발견 버그 수
  → 감소 추세가 목표
```

## 3. 코드 커버리지

```ts
// jest.config.ts
coverageThreshold: {
  global: {
    lines:      80,
    functions:  80,
    branches:   70,
    statements: 80,
  },
  './src/services/': {
    lines: 90,  // 서비스 레이어는 더 높게
  },
}

// 커버리지 리포트 해석
// Lines:     실행된 코드 라인
// Functions: 호출된 함수 수
// Branches:  if/else 분기 중 실행된 비율
// Statements:실행된 구문 수 (lines와 유사하지만 더 세분화)
```

## 4. 메트릭 대시보드

```
Jira/Linear 대시보드:
  - 스프린트별 버그 추이 그래프
  - 심각도별 오픈 버그 현황
  - 평균 해결 시간 트렌드

CI/CD 메트릭:
  - 테스트 성공률 (주간 추이)
  - 빌드 시간 추이
  - Flaky 테스트 목록

커버리지 대시보드 (Codecov/SonarQube):
  - 전체/모듈별 커버리지
  - PR 별 커버리지 변화량
```

## 5. 메트릭 기반 의사결정

```
시나리오 1: 운영 버그 급증
  원인 분석:
    - E2E 테스트 커버리지 부족?
    - 특정 모듈에 집중 발생?
    - 최근 대규모 리팩토링 후?
  대응:
    - 해당 모듈 회귀 테스트 강화
    - 탐색적 테스트 세션 추가

시나리오 2: CI 테스트 시간 30분 초과
  원인 분석:
    - E2E 테스트 비중이 높다?
    - Flaky 테스트로 인한 재시도?
    - DB 세팅 시간이 길다?
  대응:
    - E2E 병렬화 (샤딩)
    - Flaky 테스트 격리/수정
    - DB 연결 재사용

시나리오 3: 자동화율 40%로 낮음
  원인 분석:
    - 반복 테스트 자동화 안 됨?
    - 자동화 인프라 미비?
  대응:
    - 자동화 ROI 높은 TC 우선 선정
    - 자동화 템플릿/프레임워크 도입
```

## 6. 안티패턴

- **커버리지만 추적**: 의미 없는 테스트로 수치만 채움
- **메트릭 없는 QA 프로세스**: 개선 방향을 알 수 없음
- **좋은 지표만 공유**: 나쁜 지표도 투명하게 공유해야 개선됨
- **목표 없는 측정**: 각 지표의 목표값 없이 측정만
- **스냅샷 없는 추세 분석**: 시계열 데이터 누적이 중요
