## 1. QA의 코드 리뷰 역할

개발자 코드 리뷰와 다른 QA 관점:
- **테스트 가능성**: 이 코드가 테스트하기 쉬운가?
- **엣지 케이스**: 예외 상황 처리가 충분한가?
- **에러 처리**: 실패 시 적절한 메시지를 주는가?
- **로깅**: 디버깅에 충분한 로그가 있는가?
- **테스트 코드**: 테스트 커버리지가 충분한가?

---

## 2. QA 코드 리뷰 체크리스트

```markdown
## 기능 관점
[ ] Happy Path 로직이 요구사항과 일치하는가?
[ ] 엣지 케이스가 처리되었는가?
    - null/undefined 입력
    - 빈 배열/문자열
    - 최대값/최소값
[ ] 에러 케이스가 처리되었는가?
    - 네트워크 오류
    - DB 오류
    - 유효성 검증 실패

## 테스트 관점
[ ] 단위 테스트가 포함되었는가?
[ ] 테스트가 Happy Path만 커버하지 않는가?
[ ] 에러 케이스 테스트가 있는가?
[ ] 테스트 커버리지가 팀 기준을 충족하는가?

## 사용자 경험 관점
[ ] 에러 메시지가 사용자가 이해할 수 있는가?
[ ] 로딩 상태가 적절히 표시되는가?
[ ] 성공/실패 피드백이 명확한가?

## 보안 관점
[ ] 입력값 검증이 충분한가?
[ ] 민감 정보가 로그에 노출되지 않는가?
[ ] 인증/인가가 적절히 적용되었는가?
```

## 3. 리뷰 코멘트 예시

```ts
// ❌ 리뷰 없이 통과될 수 있는 코드
async function updateUserProfile(userId: string, data: UpdateProfileDto) {
  const user = await userRepo.findById(userId)
  return userRepo.update(userId, data)
}

// QA 리뷰 코멘트:
// 1. user가 null인 경우 처리가 없습니다.
//    findById가 null을 반환하면 어떻게 되나요?
//    404 응답이 필요하지 않을까요?

// 2. data.profileImage가 매우 큰 파일일 경우를 처리하고 있나요?
//    크기 제한 검증이 필요해 보입니다.

// 3. 이 함수에 대한 테스트가 없습니다.
//    최소한 사용자 미존재 케이스 테스트가 필요합니다.
```

## 4. 테스트 코드 리뷰

```ts
// 리뷰 시 체크할 테스트 코드 품질

// ❌ 의미 없는 테스트
it('works', () => {
  const result = calculateDiscount(100, 'VIP')
  expect(result).toBeDefined()  // 무조건 통과
})

// ✅ 의미 있는 테스트
it('VIP 등급에게 20% 할인 적용', () => {
  expect(calculateDiscount(10000, 'VIP')).toBe(8000)
})

it('음수 금액 입력 시 에러 발생', () => {
  expect(() => calculateDiscount(-1000, 'VIP')).toThrow('금액은 0 이상')
})

// 리뷰 포인트:
// - 테스트명이 의도를 명확히 설명하는가?
// - 다양한 케이스(성공/실패/경계)를 커버하는가?
// - assert가 구체적인가? (toBeDefined 대신 toBe(8000))
// - 테스트가 독립적으로 실행 가능한가?
```

## 6. 안티패턴

- **코드만 리뷰, 테스트 미리뷰**: 테스트 코드도 리뷰 필수
- **승인만 하는 리뷰**: 구체적인 개선 제안 포함
- **QA가 기능 로직 리뷰**: 로직은 개발자 리뷰, QA는 품질/테스트 관점
- **리뷰 없이 급하게 머지**: 릴리스 압박이 있어도 최소한의 리뷰
