## 1. 운영팀에서의 애자일

개발팀만의 방법론이 아님. 콘텐츠/마케팅/운영 팀에서도 적용 가능.

**핵심 원칙:**
- 짧은 사이클로 반복 (스프린트)
- 지속적 피드백과 개선
- 유연한 우선순위 조정
- 투명한 진행 상황

---

## 2. 운영팀 스프린트 구조

```
스프린트 기간: 2주 (또는 1개월, 팀 상황에 따라)

스프린트 시작:
  - 스프린트 플래닝 (1시간)
  - 백로그에서 이번 스프린트 작업 선정
  - 스토리 포인트 추정

스프린트 중:
  - 일일 스탠드업 (15분)
  - 블로킹 이슈 즉시 해결

스프린트 종료:
  - 스프린트 리뷰: 완료한 것 확인 (30분)
  - 회고 (Retrospective): 잘된 것/개선할 것 (30분)
```

---

## 3. 칸반 (Kanban) 활용

```
콘텐츠 팀 칸반 보드:

│ 백로그  │ 이번 주  │ 진행 중  │ 검토 중  │ 완료  │
│───────│─────────│────────│────────│──────│
│ 주제#5  │ 블로그#3  │ 블로그#2  │ SNS#4   │ 블로그#1 │
│ 주제#6  │ 이메일#1  │          │          │ SNS#1~3 │
│ ...     │          │          │          │          │

WIP (Work In Progress) 제한:
  "진행 중" 열에 최대 3개 → 집중도 유지, 완료 속도 향상
```

---

## 4. 회고 (Retrospective)

```
Keep-Problem-Try (KPT) 방식:

Keep (계속할 것):
  "주간 스탠드업이 팀 동기화에 효과적이었다"
  "브리프 템플릿이 작성 시간을 줄여줬다"

Problem (문제):
  "클라이언트 승인이 자주 지연된다"
  "긴급 요청이 계획을 방해한다"

Try (시도할 것):
  "클라이언트 승인 SLA를 계약서에 명시하자"
  "긴급 요청을 위한 버퍼 시간을 20% 확보하자"
```

---

## 5. 스프린트 플래닝

```
작업 크기 추정 (T-셔츠 사이즈):
  S (1~2시간): SNS 포스트 1개
  M (반나절): 블로그 포스트 1편
  L (하루): 영상 기획 + 대본
  XL (2~3일): 월간 리포트 + 전략 수립

스프린트 용량 계산:
  팀원 2명 × 1스프린트(2주) × 70% 실제 작업 = 56 작업 시간
  → S: 28개 / M: 14개 / L: 7개 중 혼합 배분
```

---

## 6. 안티패턴

- **애자일 형식만 따르기**: 스탠드업은 하지만 변화 없음 → 목적 이해 필요
- **회고 없는 스프린트**: 반복 실수 → 회고를 통한 학습 필수
- **과도한 계획**: 2주치를 세세하게 계획 → 유연성 감소
- **WIP 제한 무시**: 동시에 너무 많은 작업 → 완료 속도 저하
- **팀원 참여 없는 플래닝**: PM 혼자 계획 → 현실성 없는 일정
