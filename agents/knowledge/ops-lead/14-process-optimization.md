## 1. 프로세스 최적화 접근법

```
1. 현재 프로세스 파악 (As-Is)
2. 문제점/낭비 식별
3. 개선 방향 설계 (To-Be)
4. 파일럿 테스트
5. 전체 적용
6. 효과 측정
```

---

## 2. 낭비 식별 (린 사고)

```
8가지 낭비 (TIMWOODS):

T - Transport: 불필요한 파일 이동, 전달
I - Inventory: 과도한 작업 백로그
M - Motion: 불필요한 반복 클릭, 검색
W - Waiting: 승인 대기, 피드백 대기
O - Overproduction: 필요 이상의 보고서, 불필요한 콘텐츠
O - Over-processing: 필요 이상의 세부 작업
D - Defects: 재작업, 오류 수정
S - Skills: 잘못된 사람에게 할당된 작업
```

**시간 낭비 감사:**
```
한 주 동안 모든 작업을 15분 단위로 기록
→ 반복 작업, 대기 시간, 회의 비중 파악
→ 자동화/위임 가능한 것 식별
```

## 3. 프로세스 매핑

```
현재 콘텐츠 승인 프로세스 (문제 있음):

PM 작성 → 팀 리드 검토 (2일) → PM 수정 → 팀 리드 재검토 (1일) →
클라이언트 승인 요청 → 클라이언트 피드백 (3~5일) → PM 수정 →
클라이언트 최종 승인 (2일) → 발행

총 9~15일 소요, 승인 단계 5회

개선된 프로세스:
PM 작성 → QC 체크리스트 자체 검토 → 클라이언트 승인 요청 (1일 내 발송) →
클라이언트 승인 (2영업일 SLA) → 발행

총 3~5일 소요, 승인 단계 2회 (팀 리드 리뷰를 QC 체크리스트로 대체)
```

## 4. OKR 기반 개선 우선순위

```
효과 × 실행 가능성 매트릭스:

               실행 어려움    실행 쉬움
효과 높음    │  Big Bet    │  Quick Win  │  ← 먼저
효과 낮음    │  Skip      │  Fill-in    │

Quick Win 예시:
  - 브리프 템플릿 표준화 (2시간 구축 → 매 작업 30분 절약)
  - 슬랙 알림 룰 설정 (30분 → 알림 노이즈 70% 감소)

Big Bet 예시:
  - 전체 워크플로우 새 도구로 이관 (1달 구축 → 연간 100시간 절약)
```

## 5. 개선 측정 방법

```
Before/After 비교:
  납품 소요 시간: 평균 5일 → 3.5일 (30% 감소)
  재작업 비율: 30% → 12%
  클라이언트 만족도: 3.8 → 4.3

측정 지표 설정:
  - 사이클 타임: 착수 → 완료 시간
  - 처리량: 주당 완료 작업 수
  - 품질: 재작업 비율, 오류율
  - 만족도: 클라이언트/팀원 NPS
```

---

## 6. 안티패턴

- **개선 없는 문제 인식**: "바쁘니까 나중에" → 프로세스 계속 악화
- **측정 없는 개선**: 개선 효과를 알 수 없음
- **팀 참여 없는 프로세스 변경**: 현장 경험이 없는 개선은 실패
- **완벽한 프로세스 추구**: 80% 완성도로 먼저 적용, 반복 개선
- **도구 교체만으로 해결 시도**: 프로세스 문제 + 도구 문제 구분
