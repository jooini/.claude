## 1. 팀 코디네이션의 핵심

각 팀원이 무엇을 하는지, 어디서 막혔는지, 무엇이 필요한지를 빠르게 파악하고 조율.

---

## 2. 일일 스탠드업

```
형식: 슬랙 또는 짧은 화상 미팅 (15분 이내)

각자 공유:
1. 어제 완료한 것
2. 오늘 할 것
3. 블로커 (없으면 없음)

예시:
홍길동: 어제 A사 3월 리포트 초안 완료. 오늘 B사 콘텐츠 브리프 작성 예정.
        블로커: C사 이미지 소스 미수령 — 디자인팀 팔로업 필요.
```

---

## 3. 리소스 관리

```
주간 리소스 현황 파악:

| 팀원 | 이번 주 가용 시간 | 현재 할당 | 남은 여유 |
|------|--------------|--------|--------|
| 홍길동 | 40h | 35h | 5h |
| 김철수 | 40h | 42h | -2h (과부하!) |
| 이영희 | 32h (목요일 반차) | 28h | 4h |

과부하 발생 시:
  1. 작업 우선순위 재조정
  2. 기한 조정 (클라이언트 협의)
  3. 타 팀원 이관
  4. 외부 지원 투입
```

---

## 4. 의존성 관리

```
작업 간 의존 관계 파악:

블로그 포스트 발행 의존성:
  작가 초안 완성
    ↓ (기한: 3/12)
  에디터 검토
    ↓ (기한: 3/13)
  디자이너 이미지 제작
    ↓ (기한: 3/14, 에디터 검토 완료 후 가능)
  클라이언트 승인
    ↓ (기한: 3/15)
  발행 (3/18)

의존성 관리 핵심:
  - 상류 작업 지연 → 하류 작업 영향 즉시 파악
  - 임박한 의존성은 미리 알림
  - 병렬 가능한 작업은 동시 진행
```

## 5. 협업 도구 설정

```
채널 구조 (슬랙 기준):

#general         — 팀 공지, 일반 소통
#daily-standup   — 일일 스탠드업
#client-a        — 클라이언트별 채널 (내부)
#client-a-ext    — 클라이언트와 공유 채널
#ops-alerts      — 긴급 이슈
#random          — 업무 외 소통

파일 관리:
  Google Drive 폴더 구조:
  📁 클라이언트/
    📁 A사/
      📁 계약/
      📁 브랜드가이드/
      📁 2024년 콘텐츠/
        📁 01월/
        📁 02월/
```

## 6. 안티패턴

- **사일로 운영**: 각자 알아서 → 조율 없이 중복/누락 발생
- **구두 커뮤니케이션만**: 기록 없음 → 책임 소재 불명확
- **모든 것을 PM이 처리**: 팀원 자율성 없음 → 병목
- **리소스 현황 파악 없음**: 과부하 팀원 방치
- **의존성 미파악**: 상류 지연이 하류에 미치는 영향 뒤늦게 인지
