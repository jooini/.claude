## 1. 대시보드 설계 원칙

- **목적 중심**: 누가, 어떤 결정을 위해 사용하는가?
- **적은 것이 많은 것**: 핵심 5~7개 지표만
- **행동 유도**: 수치만이 아닌 "좋다/나쁘다" 기준 포함
- **실시간 또는 자동 업데이트**: 수동 업데이트 대시보드는 구식이 됨

---

## 2. 계층별 대시보드

```
경영진 대시보드 (월간, 1페이지)
  → 클라이언트 수, 매출, 이탈율, 평균 만족도

운영 대시보드 (주간, 2~3페이지)
  → 납품 현황, SLA 준수율, 팀 리소스, 이슈 현황

클라이언트 대시보드 (주간/월간)
  → 채널별 성과, 콘텐츠 현황, 목표 대비
```

## 3. Google Looker Studio 구성

```
데이터 소스 연결:
  Google Analytics 4 → 웹 트래픽
  Google Search Console → SEO 키워드
  Google Sheets → 수동 데이터, KPI 목표값
  Meta Insights → SNS 성과
  Google Ads → 광고 성과

주요 컴포넌트:
  Scorecard: 핵심 지표 현재값 + 전월 대비
  Time Series: 추세 확인
  Bar Chart: 채널/콘텐츠별 비교
  Table: 상세 데이터

필터:
  날짜 범위 선택
  클라이언트 선택 (멀티 클라이언트 관리 시)
```

## 4. 클라이언트 대시보드 예시 구조

```
섹션 1: 이번 달 핵심 수치
  오가닉 트래픽: 14,600 (목표 13,000 ✅ 112%)
  전환: 52건 (목표 40건 ✅ 130%)
  인게이지먼트율: 4.2% (목표 3.5% ✅)

섹션 2: 채널별 성과 추이 (최근 6개월)
  [라인 차트: 블로그 트래픽 추이]
  [라인 차트: SNS 팔로워 추이]

섹션 3: 콘텐츠 현황
  이번 달 발행: 12개 (계획 12개 ✅)
  다음 달 예정: 14개

섹션 4: 키워드 순위 현황
  [테이블: 목표 키워드별 현재 순위]
```

## 5. KPI 목표값 설정

```
SMART 목표:
  S - Specific: 오가닉 트래픽 (막연한 "트래픽" X)
  M - Measurable: GA4로 측정 가능
  A - Achievable: 현실적 기준 (전월 +20%가 아닌 +10~15%)
  R - Relevant: 비즈니스 목표와 연결
  T - Time-bound: 3월 말까지

좋은 KPI 예시:
  2024년 3월 목표: 오가닉 트래픽 13,000 (2월 11,200 대비 +16%)
나쁜 KPI 예시:
  "트래픽 많이 늘리기"
```

---

## 6. 안티패턴

- **지표 과다**: 50개 KPI → 핵심 5~7개로
- **목표 없는 지표**: 숫자만 있고 기준 없음 → "좋다/나쁘다" 판단 불가
- **수동 업데이트 대시보드**: 구식 데이터 → 자동화
- **클라이언트에게 맞지 않는 대시보드**: 경영진용을 실무담당자에게
- **대시보드만 공유, 해석 없음**: 수치 + 인사이트 함께
