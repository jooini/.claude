## 1. 온보딩의 목적

계약 이후 실제 협업이 원활하게 시작되도록.
첫 30일이 장기 관계의 기반을 결정한다.

**좋은 온보딩의 효과:**
- 기대치 불일치로 인한 갈등 예방
- 업무 속도 빠른 정상화
- 클라이언트 신뢰 조기 확보

---

## 2. 온보딩 체크리스트

```markdown
## 계약 직후 (Day 0~3)

[ ] 킥오프 미팅 일정 잡기
[ ] 계약서/SOW 최종 확인
[ ] 내부 팀 브리핑 (담당자, 역할, 일정)
[ ] 프로젝트 채널 개설 (슬랙, 이메일 등)
[ ] 클라이언트 정보 수집 폼 발송

## 킥오프 미팅 (Day 3~7)

[ ] 팀 소개
[ ] 목표 및 KPI 합의
[ ] 작업 프로세스/워크플로우 설명
[ ] 커뮤니케이션 방식 합의 (채널, 응답 시간, 담당자)
[ ] 승인 프로세스 확인
[ ] 리스크 및 제약 사항 공유

## 초기 셋업 (Day 7~14)

[ ] 브랜드 가이드, 자산 수령
[ ] 기존 자료 이관 (과거 성과 데이터, 콘텐츠 등)
[ ] 필요 접근 권한 부여 (GA, 광고 계정 등)
[ ] 첫 번째 작업물 납품 계획 공유

## 안정화 (Day 14~30)

[ ] 첫 번째 납품물 전달
[ ] 피드백 수렴 및 반영
[ ] 프로세스 조정
[ ] 30일 리뷰 미팅 진행
```

## 3. 킥오프 미팅 아젠다

```markdown
# 킥오프 미팅 아젠다

시간: 90분
참석: 클라이언트 의사결정자 + 실무자, 내부 팀

1. 팀 소개 (10분)
   - 각자 역할과 연락처 교환

2. 목표 정렬 (20분)
   - "이 프로젝트가 성공하면 어떤 모습인가요?"
   - KPI 합의 (정량적 목표)
   - 우선순위 합의

3. 현황 파악 (20분)
   - 현재 어디에 있는지
   - 가장 큰 페인 포인트
   - 과거 시도한 것과 결과

4. 작업 방식 (20분)
   - 타임라인 공유
   - 승인 프로세스 (누가, 몇 일 내)
   - 커뮤니케이션 채널/빈도
   - 피드백 방식

5. Q&A (10분)
6. 다음 단계 확인 (10분)
```

## 4. 클라이언트 정보 수집 폼

```markdown
## 온보딩 정보 수집 — [클라이언트명]

### 기본 정보
- 회사 소개 (한 문단)
- 주요 제품/서비스
- 타겟 고객 (페르소나)
- 경쟁사 (알고 있는 것)

### 목표와 KPI
- 이번 계약을 통해 달성하고 싶은 것 (TOP 3)
- 성공의 기준 (측정 가능한 수치로)
- 기간: ___월 기준

### 브랜드 가이드라인
- 브랜드 톤앤매너
- 피해야 할 표현/이미지
- 선호하는 참고 사례 (경쟁사 외)

### 접근 권한 필요 목록
- Google Analytics
- 광고 계정 (Meta, Google)
- CMS / 블로그 관리자
- SNS 계정
```

## 5. 기대치 관리

```
초기에 명확히 해야 할 것:

✅ 무엇을 하는가 (계약 범위)
✅ 무엇을 하지 않는가 (범위 외)
✅ 어떤 결과를 기대할 수 있는가 (현실적 KPI)
✅ 어떤 결과는 보장 못하는가 (외부 변수)
✅ 클라이언트가 해야 할 것 (피드백, 자료 제공)
✅ 변경 요청 처리 방식 (추가 비용, 일정 조정)
```

---

## 6. 안티패턴

- **킥오프 없이 바로 작업 시작**: 방향 불일치로 재작업 발생
- **목표 수치 합의 없음**: "더 많은 트래픽" → "3개월 내 오가닉 트래픽 30% 증가"
- **승인 프로세스 불명확**: 누가 최종 승인자인지 모름
- **클라이언트 역할 미정의**: 자료 제공 지연이 일정 지연으로 이어짐
- **첫 납품 너무 늦음**: 첫 2주 내 작은 성과물이라도 전달
