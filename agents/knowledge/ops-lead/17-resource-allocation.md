## 1. 리소스 유형

```
인적 리소스:   팀원, 프리랜서, 외부 전문가
시간 리소스:   가용 업무 시간
예산 리소스:   외주, 도구, 광고비
도구 리소스:   소프트웨어, 구독 서비스
```

---

## 2. 리소스 계획 프로세스

```
1. 프로젝트 요구사항 파악
   → 어떤 스킬이 얼마나 필요한가?

2. 현재 리소스 현황 파악
   → 팀원별 가용 시간, 스킬 매핑

3. 갭 분석
   → 부족한 것: 외주 또는 채용
   → 과잉인 것: 타 프로젝트 배정

4. 배정
   → 적합한 사람에게 적합한 작업

5. 모니터링
   → 과부하/미달 지속 모니터링
```

## 4. 리소스 배분 원칙

```
70-20-10 법칙:
  70% — 핵심 업무 (계약된 클라이언트 작업)
  20% — 개선 활동 (프로세스 개선, 새 기술 학습)
  10% — 실험/혁신 (새 접근 방식 테스트)

버퍼 확보:
  가용 시간의 20%는 긴급 요청, 예상치 못한 작업을 위해 비워두기
  100% 배정 = 첫 돌발 상황에서 전체 지연
```

---

## 5. 과부하 감지 및 해결

```
과부하 신호:
  - 마감 지연 빈도 증가
  - 품질 저하
  - 야근 증가
  - 팀원 번아웃 징조

해결 방법 (우선순위 순):
  1. 우선순위 재조정 (덜 중요한 것 미루기)
  2. 기한 협상 (클라이언트와)
  3. 작업 재배분 (다른 팀원에게)
  4. 외주 활용 (프리랜서)
  5. 추가 채용 검토
```

---

## 6. 안티패턴

- **스킬 무관한 배정**: "누가 비어있나"만 보고 배정
- **100% 가동율 목표**: 버퍼 없음 → 한 번의 변수에 전체 지연
- **리소스 현황 미파악**: 뒤늦게 과부하 발견
- **외주 비용 미고려**: 예산 계획 없이 외주 → 비용 초과
- **단일 의존 리소스**: 특정 기술을 1명만 → SPOF
