## 1. 이해관계자 유형별 커뮤니케이션

```
C레벨 / 경영진
  → 핵심 수치 + 비즈니스 임팩트만
  → 주간/월간 1~2페이지 요약

중간 관리자 / 팀 리드
  → 진행 상황 + 이슈 + 의사결정 필요 사항
  → 주간 업데이트

실무 담당자
  → 세부 작업 현황, 기술적 세부사항
  → 일간/수시 업데이트

외부 파트너/클라이언트
  → 합의된 형식으로 정기 보고
  → 주간/월간 리포트
```

## 2. 경영진 업데이트 (Executive Summary)

```markdown
# 3월 운영 현황 — 경영진 업데이트

## 핵심 현황 (한 눈에)
✅ 목표 달성: 클라이언트 A, B 월간 KPI 초과 달성
⚠️ 주의 필요: 클라이언트 C — 트래픽 목표 85% 달성 (원인 분석 중)
🔴 이슈: 신규 클라이언트 D — 온보딩 1주 지연 (킥오프 일정 조율 중)

## 수치 요약
- 관리 클라이언트: 8개사
- 이번 달 납품 건수: 47건 (목표 45건 ✅)
- 평균 클라이언트 만족도: 4.3/5.0
- 계약 갱신율 (이번 분기): 87.5%

## 다음 달 주요 일정
- 클라이언트 D 온보딩 완료 (4/5 예정)
- 클라이언트 E 계약 갱신 협의 (4/15)
- 팀 증원 1명 (4월 입사 예정)

## 의사결정 필요
클라이언트 C의 추가 지원 여부 → 4/3 미팅에서 논의 요청
```

## 3. 정기 업데이트 리듬

```
일간 (팀 내부):
  - 슬랙 스탠드업: 오늘 할 것, 어제 한 것, 블로커

주간 (팀 + 클라이언트):
  - 주간 업무 현황 보고
  - 이슈/리스크 공유
  - 다음 주 계획

월간 (경영진 + 클라이언트):
  - 성과 리포트
  - 목표 대비 현황
  - 다음 달 전략
```

## 4. 나쁜 소식 전달하기

```
SCQA 프레임워크:

Situation (상황):
  "현재 클라이언트 C의 3월 오가닉 트래픽이 목표 대비 85% 수준입니다."

Complication (문제):
  "이 추세가 지속되면 분기 KPI 달성이 어려울 수 있습니다."

Question (질문):
  "어떻게 대응해야 할까요?"

Answer (답변):
  "저희는 4월 전략을 SEO 집중형으로 전환하고,
   추가로 백링크 구축 캠페인을 병행할 것을 제안합니다.
   이를 위해 예산 20% 재배분이 필요합니다."
```

## 5. 이해관계자 지도 만들기

```
이해관계자 분류:

영향력 높음 + 관심도 높음 → 긴밀히 관리 (정기 1:1, 세부 업데이트)
영향력 높음 + 관심도 낮음 → 만족 유지 (요약 보고, 이슈만 공유)
영향력 낮음 + 관심도 높음 → 정보 제공 (정기 뉴스레터, 공개 보고)
영향력 낮음 + 관심도 낮음 → 모니터링만 (최소 커뮤니케이션)
```

## 6. 안티패턴

- **모든 이해관계자에게 동일한 업데이트**: 레벨/관심사에 맞게 맞춤화
- **좋을 때만 보고**: 나쁜 소식도 선제적으로 → 신뢰 구축
- **숫자 없는 업데이트**: "잘 되고 있습니다" → 구체적 수치
- **액션 없는 문제 보고**: 문제 + 원인 + 해결책 함께
- **업데이트 빈도 너무 낮음**: 경영진이 현황 모르면 불필요한 개입
