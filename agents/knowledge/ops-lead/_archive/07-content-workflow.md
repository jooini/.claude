## 1. 콘텐츠 워크플로우 설계 목적

- 작업 상태 가시화 (누가, 무엇을, 어디서 막혔는지)
- 납기 예측 가능성 향상
- 병목 지점 파악 및 제거
- 팀원 간 혼선 감소

---

## 2. 기본 콘텐츠 워크플로우

```
기획 → 브리프 작성 → 작성 → 1차 검토 → 수정
→ QC → 클라이언트 승인 → 발행 → 성과 모니터링
```

### 단계별 정의

| 단계 | 담당 | 산출물 | 기한 |
|------|------|------|------|
| 기획 | PM | 월간 콘텐츠 계획 | 전월 25일 |
| 브리프 | PM | 콘텐츠 브리프 | D-7 |
| 작성 | 작가 | 초안 | D-4 |
| 1차 검토 | 에디터 | 검토 피드백 | D-3 |
| 수정 | 작가 | 수정본 | D-2 |
| QC | QC 담당 | 최종 검토 | D-1 |
| 승인 | 클라이언트 | 승인 | D-1 |
| 발행 | 운영팀 | 발행 완료 | D-day |

## 3. 콘텐츠 브리프 템플릿

```markdown
# 콘텐츠 브리프

## 기본 정보
- 제목(안): [제목]
- 콘텐츠 유형: 블로그 포스트 / SNS / 이메일
- 목표 키워드: [키워드] (월 검색량: )
- 발행 예정일: 

## 목적
이 콘텐츠로 달성하고자 하는 것:
- 타겟 독자:
- 독자가 얻어가야 할 것:
- 원하는 행동(CTA):

## 내용 가이드
- 핵심 메시지 (1~2문장):
- 포함해야 할 내용:
  1. 
  2. 
  3.
- 제외해야 할 내용:
- 참고 자료/링크:

## 형식 요구사항
- 분량: 약 ___ 자
- 톤: 전문적 / 친근한 / 중립적
- 이미지 필요 여부: Y / N
- SEO 요구사항: H1, H2 포함, 내부 링크 2개 이상
```

## 4. 작업 관리 도구 활용

```
Notion 칸반 보드 예시:

| 기획 | 작성 중 | 검토 중 | 승인 대기 | 발행 완료 |
|-----|--------|--------|---------|---------|
| 4월 블로그 #1 | 4월 블로그 #2 | 3월 SNS #5 | 3월 블로그 #3 | 3월 블로그 #1 |
| 이메일 뉴스레터 | | | | 3월 SNS #1~4 |

각 카드에 포함:
- 담당자
- 기한
- 현재 단계
- 블로킹 이슈
```

---

## 5. 병목 해결

```
흔한 병목 지점:

1. 클라이언트 승인 지연
   해결: 승인 기한 SLA 명시 (예: "2영업일 내 승인 요청")
         기한 내 무응답 = 승인으로 처리 (사전 합의 시)

2. 피드백 방향 불명확
   해결: 수정 요청 시 구체적 방향 필수
         "좀 더 자연스럽게" → "3단락 문장을 나눠서 읽기 쉽게"

3. 작가 블로킹 (브리프 불충분)
   해결: 브리프 표준화, 착수 전 질문 정리 시간 확보

4. QC 백로그 쌓임
   해결: QC 담당자 전담 또는 요일별 QC 시간 고정
```

## 6. 안티패턴

- **워크플로우 문서화 없음**: 구두로만 → 새 팀원 온보딩 어려움
- **단일 담당자 병목**: 한 사람이 모든 것을 → 휴가/이탈 시 마비
- **상태 업데이트 없음**: 작업 상태 불투명 → PM이 일일이 확인
- **버전 관리 없음**: 어떤 것이 최신 파일인지 혼란
- **발행 후 모니터링 없음**: 성과 확인 없이 다음 콘텐츠로
