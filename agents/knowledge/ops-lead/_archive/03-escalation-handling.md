## 1. 에스컬레이션이란

현재 담당자 레벨에서 해결 불가한 이슈를 상위 권한자에게 넘기는 것.
빠른 에스컬레이션이 문제를 키우지 않는다.

---

## 2. 에스컬레이션 기준

```
즉시 에스컬레이션 (당일):
  - 클라이언트가 계약 해지를 언급
  - 납품 기한 2일 이상 지연이 불가피
  - 팀 내 해결 불가한 기술적 이슈
  - 클라이언트 불만이 고조되고 있음

24시간 내 에스컬레이션:
  - 예산/범위 변경 요청
  - 내부 리소스 부족으로 일정 영향
  - 계약서 해석에 이견 발생
  - 반복되는 피드백 사이클로 진행 막힘
```

---

## 3. 에스컬레이션 프로세스

```
1단계: 담당 PM이 자체 해결 시도
       - 클라이언트와 직접 논의
       - 내부 리소스 조율

2단계: 팀 리드에게 에스컬레이션
       - 상황 공유 + 시도한 것 설명
       - 필요한 지원 요청

3단계: 경영진 개입
       - 계약 관련 의사결정 필요
       - 클라이언트 C레벨 대응 필요
       - 재무적 영향 있는 결정
```

---

## 4. 에스컬레이션 보고 형식

```markdown
## 에스컬레이션 보고

일시: 2024-03-15 14:00
작성자: 홍길동 PM
대상: 팀 리드 김OO

### 상황 요약
클라이언트 A사 김부장님이 3월 콘텐츠 성과에 강한 불만을 표시하며
계약 재검토를 언급하셨습니다.

### 배경
- 3월 블로그 트래픽: 전월 대비 -5% (목표 +20%)
- 원인: 구글 알고리즘 업데이트 영향 (업계 전반 하락)
- 지난 2주간 3차례 설명했으나 납득 못하고 계심

### 시도한 것
- 업계 데이터 공유 (경쟁사도 동일 하락)
- 알고리즘 업데이트 관련 외부 리포트 공유
- 4월 회복 전략 제시

### 필요한 지원
- 팀 리드 또는 대표가 클라이언트와 직접 미팅
- 필요 시 계약 조건 조정 논의

### 제안 액션
이번 주 내 3자 미팅 (클라이언트, 팀 리드, PM) 요청
```

## 5. 클라이언트 불만 대응

```
불만 대응 5단계:

1. 경청 — 충분히 듣기 (방어적으로 반응하지 않기)
2. 공감 — "그렇게 느끼셨군요, 이해합니다"
3. 인정 — 잘못이 있으면 명확히 인정
4. 해결책 제시 — 구체적인 액션과 기한
5. 팔로우업 — 해결 후 확인

대화 예시:
클라이언트: "3개월째 성과가 없잖아요. 이게 뭐 하는 건지 모르겠어요."

PM: "말씀하신 부분 충분히 이해합니다. 기대하셨던 성과가 나오지 않아
    답답하셨을 것 같습니다. (공감)

    지난 3개월 성과를 데이터로 같이 검토해보고 싶습니다.
    일부 지표는 개선됐지만 핵심 목표에 미달한 부분이 있어
    솔직하게 논의드리고 싶습니다. (인정)

    이번 주 안으로 미팅을 잡고, 남은 기간 전략을 조정하는 방안을
    구체적으로 제안드리겠습니다. (해결책)

    언제 시간이 가능하신지요?"
```

## 6. 안티패턴

- **에스컬레이션 지연**: 문제가 커진 후 보고 → 선제적으로
- **변명부터 시작**: 이유 설명 전 공감과 인정 먼저
- **에스컬레이션 없이 혼자 처리**: 권한 밖 약속 → 더 큰 문제
- **에스컬레이션 후 방치**: 상위 개입 후에도 PM이 팔로우업 책임
- **감정적 대응**: 클라이언트 분노에 같이 감정적으로 반응
