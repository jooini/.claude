## 1. 콘텐츠 QC란

발행 전 콘텐츠 품질을 체계적으로 검토하는 프로세스.
"눈에 보이는 실수"를 잡는 것을 넘어 "목적에 맞는가"를 검증.

---

## 2. QC 체크리스트

### 공통 (모든 콘텐츠)

```markdown
## 기본 품질
[ ] 맞춤법/문법 오류 없음 (맞춤법 검사기 통과)
[ ] 사실 확인 (수치, 날짜, 인명 등)
[ ] 브랜드 톤앤매너 일치
[ ] 금지 표현/경쟁사 언급 없음

## 브랜드 일관성
[ ] 브랜드 가이드라인 준수
[ ] 폰트, 색상, 로고 규격 준수 (시각 콘텐츠)
[ ] 제품/서비스명 표기 일치 (대소문자, 띄어쓰기)

## 법적/윤리적
[ ] 저작권 침해 없음 (이미지, 인용)
[ ] 개인정보 포함 여부 확인
[ ] 광고 표시 의무 준수 (광고/협찬인 경우)
[ ] 과장/허위 광고 해당 여부 확인
```

### 블로그 포스트

```markdown
[ ] 목표 키워드 포함 (제목, 첫 문단, 소제목)
[ ] 메타 타이틀 60자 이내
[ ] 메타 디스크립션 160자 이내
[ ] 이미지 alt 텍스트 포함
[ ] 내부 링크 2개 이상
[ ] CTA(Call to Action) 포함
[ ] 가독성: 짧은 단락, 소제목 활용
```

### SNS 콘텐츠

```markdown
[ ] 플랫폼별 글자 수 제한 확인
[ ] 해시태그 적절성 (개수, 관련성)
[ ] 링크 미리보기 확인
[ ] 이미지 사이즈 규격 (플랫폼별)
[ ] 게시 시간 최적화 고려
```

## 3. QC 프로세스

```
작성자 1차 자체 검토
    ↓
에디터/QC 담당자 검토 (오탈자, 사실, 브랜드)
    ↓
PM 최종 확인 (클라이언트 기준 적합성)
    ↓
클라이언트 승인 (필요 시)
    ↓
발행
```

**QC 역할 분리 원칙:**
- 작성자 ≠ 최종 검토자 (본인 글은 오류 인지 어려움)
- 최소 2인 이상 검토 (중요 콘텐츠)

## 4. 자동화 QC 도구

```
맞춤법/문법:
  - 국립국어원 맞춤법 검사기
  - 네이버 맞춤법 검사기
  - Grammarly (영문)

가독성:
  - Hemingway Editor (영문)
  - 문장 길이, 수동태 과다 확인

SEO 검토:
  - Yoast SEO (WordPress)
  - Surfer SEO
  - 목표 키워드 밀도 확인

이미지:
  - 규격 확인: 플랫폼별 권장 사이즈
  - 용량 압축: TinyPNG, Squoosh
```

## 5. QC 피드백 작성법

```
❌ 나쁜 피드백:
"이 부분 이상해요."
"전체적으로 다시 써주세요."

✅ 좋은 피드백:
"3단락 2문장 — '획기적인 혁신'은 과장 표현으로
 브랜드 가이드라인 톤에 맞지 않습니다.
 '새로운 접근 방식' 정도로 수정해주세요."

피드백 형식:
  위치 + 현재 내용 + 문제점 + 수정 방향
```

---

## 6. 안티패턴

- **QC 없이 발행**: 오탈자 하나가 브랜드 신뢰 손상
- **작성자 자체 QC만**: 본인 글의 오류 인지 어려움
- **체크리스트 없는 QC**: 매번 다른 기준 → 일관성 없음
- **피드백 없는 반려**: "다시 해주세요"만 → 개선 방향 불명확
- **QC 시간 미확보**: 마감 직전 QC → 빠른 검토로 실수 놓침
