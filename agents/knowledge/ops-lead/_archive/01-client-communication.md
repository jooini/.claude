## 1. 클라이언트 커뮤니케이션 원칙

- **투명성**: 좋은 소식만 아니라 나쁜 소식도 즉시 공유
- **선제적**: 클라이언트가 묻기 전에 먼저 업데이트
- **구체성**: "빠르게 처리하겠습니다" 대신 "내일 오후 3시까지 완료"
- **책임감**: 문제 발생 시 변명보다 해결책 먼저

---

## 2. 커뮤니케이션 채널별 기준

| 채널 | 용도 | 응답 기준 |
|------|------|----------|
| 슬랙/메신저 | 일상 업무, 빠른 확인 | 업무 시간 내 1~2시간 |
| 이메일 | 공식 요청, 문서 공유 | 24시간 이내 |
| 화상/전화 | 복잡한 논의, 이슈 해결 | 사전 예약 |
| 미팅 | 정기 리뷰, 전략 논의 | 주간/격주 |

---

## 3. 정기 업데이트 템플릿

```markdown
# 주간 업무 업데이트 — {날짜}

## ✅ 완료
- 랜딩 페이지 A/B 테스트 설계 완료
- 3월 콘텐츠 캘린더 초안 공유 (검토 요청)

## 🔄 진행 중
- 블로그 포스트 4편 작성 중 (3/4 완료, 금요일 납품 예정)
- SNS 광고 소재 제작 중 (디자인팀 검토 단계)

## ⚠️ 이슈 / 리스크
- 키워드 조사 결과 경쟁 강도 예상보다 높음
  → 전략 조정 필요, 목요일 미팅에서 논의 요청

## 📅 다음 주 계획
- 블로그 포스트 최종 납품
- 4월 캘린더 기획 시작
- 광고 소재 런칭 (수요일 예정)

## 📌 확인 필요
- 3월 콘텐츠 캘린더 승인 → 이번 주 수요일까지 피드백 주시면 감사합니다
```

## 4. 어려운 상황 커뮤니케이션

### 지연 발생 시

```
❌ 나쁜 예:
"작업이 좀 늦어지고 있습니다. 최대한 빨리 해드리겠습니다."

✅ 좋은 예:
"안녕하세요, 블로그 포스트 납품 일정을 공유드립니다.
당초 목요일 납품 예정이었으나, [구체적 이유]로 인해
금요일 오후 3시로 조정이 필요합니다.

불편을 드려서 죄송합니다. 품질에는 영향이 없도록 하겠습니다.
혹시 일정 조정이 어려우시면 말씀해 주세요, 방법을 찾겠습니다."
```

### 클라이언트 요청 거절 시

```
✅ 공감 → 이유 → 대안 순서로:

"말씀하신 방향 이해합니다. [요청 내용 요약]은
현재 계약 범위 밖으로, 추가 진행 시 [비용/일정] 조정이 필요합니다.

대신 현재 범위 안에서 [대안]으로 비슷한 효과를 낼 수 있을 것 같습니다.
어떻게 생각하시나요?"
```

## 5. 미팅 운영

**미팅 전:**
- 아젠다 최소 24시간 전 공유
- 필요 자료 미리 배포

**미팅 중:**
- 시간 엄수 (시작/종료)
- 액션 아이템 실시간 기록
- 결정 사항 재확인

**미팅 후:**
- 24시간 내 미팅 노트 공유
- 담당자/기한 명시된 액션 아이템

```markdown
# 미팅 노트 — {날짜} {제목}

참석자: 클라이언트(김OO, 박OO), 담당 PM(홍길동)

## 논의 내용
1. 3월 성과 리뷰 — 목표 대비 120% 달성
2. 4월 전략 방향 합의 — 검색 최적화 집중

## 결정 사항
- 4월 블로그 주제: SEO 중심으로 전환
- 예산 10% 증액 승인

## 액션 아이템
| 항목 | 담당 | 기한 |
|------|------|------|
| 4월 키워드 리서치 | 홍길동 | 3/15 |
| 예산 승인 서류 | 클라이언트 박OO | 3/13 |

## 다음 미팅
3/22 (화) 오전 10시
```

## 6. 안티패턴

- **문제 숨기기**: 나중에 더 큰 신뢰 손상
- **과도한 약속**: 지킬 수 없는 기한 제시
- **일방적 보고**: 클라이언트 의견 반영 없는 통보
- **응답 지연**: 확인만 해도 되는 상황에서 무응답
- **모호한 표현**: "곧", "빠르게", "최대한" → 구체적 날짜와 수치로
