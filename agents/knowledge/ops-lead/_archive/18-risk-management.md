## 1. 리스크 관리 프로세스

```
식별 → 평가 → 대응 계획 → 모니터링 → 실행
```

---

## 2. 리스크 식별

```
일반적인 운영 리스크:

인력 리스크:
  - 핵심 팀원 갑작스러운 이탈
  - 스킬 부족으로 품질 저하
  - 번아웃으로 생산성 저하

클라이언트 리스크:
  - 갑작스러운 계약 해지
  - 비현실적 요구사항 변경
  - 승인 지연으로 납기 영향

프로세스 리스크:
  - 도구 장애 (GA, 작업 관리 도구)
  - 외부 플랫폼 정책 변경 (구글 알고리즘 등)
  - 납품 오류/품질 문제

재무 리스크:
  - 미수금 발생
  - 예산 초과
  - 클라이언트 대금 지연
```

## 3. 리스크 평가 매트릭스

```
리스크 = 발생 가능성 × 영향도

                낮음(1)     중간(2)     높음(3)
발생 가능성 높음(3)│  3      │   6      │   9    │ ← 최우선
발생 가능성 중간(2)│  2      │   4      │   6    │
발생 가능성 낮음(1)│  1      │   2      │   3    │

리스크 레지스터:
| 리스크 | 가능성 | 영향 | 점수 | 대응 방안 | 담당 |
|--------|------|------|------|---------|------|
| 핵심 작가 이탈 | 2 | 3 | 6 | 프리랜서 풀 3명 유지 | PM |
| 구글 알고리즘 변경 | 3 | 2 | 6 | 다채널 전략, 빠른 대응 | 전략팀 |
| 클라이언트 미수금 | 1 | 3 | 3 | 선금 조건 협상 | 재무 |
```

---

## 4. 대응 전략

```
회피 (Avoid): 리스크 원인 제거
  단일 클라이언트 의존도 70% 이상 → 신규 클라이언트 확보로 분산

감소 (Reduce): 발생 가능성/영향 축소
  핵심 지식 문서화 → 팀원 이탈 시 지식 보존

이전 (Transfer): 타 주체에게 이전
  특정 작업 리스크 → 계약서에 클라이언트 책임 명시
  재무 리스크 → 보험 가입

수용 (Accept): 비용 대비 수용
  소규모 일정 지연 → 버퍼로 흡수
```

## 5. 비상 계획 (Contingency Plan)

```
시나리오: 핵심 작가 갑작스러운 이탈

즉각 조치 (D-day):
  1. 현재 진행 중인 작업 목록 파악
  2. 납기 임박 작업 우선 처리
  3. 클라이언트 상황 공유 (솔직하게)

단기 조치 (1주 내):
  4. 프리랜서 긴급 투입 (풀 유지 중)
  5. 타 팀원 임시 지원
  6. 납기 조정 협의

중기 조치 (1개월):
  7. 대체 채용 또는 프리랜서 계약
  8. 지식 이전 문서 강화 (재발 방지)
```

---

## 6. 안티패턴

- **리스크 관리 = 문제 발생 후 대응**: 사전 파악과 예방이 핵심
- **문서에만 존재하는 리스크 레지스터**: 정기 검토 없음
- **모든 리스크를 동일하게**: 우선순위화 없이 모든 것을 관리하려다 아무것도 못함
- **SPOF 방치**: 한 사람/도구에 의존하는 구조 그대로
- **비상 계획 없는 리스크 식별**: "알고 있다"는 것만으로 충분하지 않음
