## 1. 프로젝트 플래닝 프레임워크

```
1. 범위 정의 (Scope)

5. 의존성 파악 (Dependencies)
6. 커뮤니케이션 계획 (Communication)
```

## 2. WBS (Work Breakdown Structure)

```
프로젝트: 브랜드 웹사이트 콘텐츠 리뉴얼

1. 전략 수립
   1.1 현황 감사 (기존 콘텐츠 인벤토리)
   1.2 경쟁사 분석
   1.3 키워드 전략 수립
   1.4 콘텐츠 전략 문서 작성

2. 콘텐츠 제작
   2.1 홈페이지 (메인 카피)
   2.2 서비스 페이지 (5개)
   2.3 About Us 페이지
   2.4 블로그 초기 포스트 (10개)

3. 검토 및 발행
   3.1 내부 QC
   3.2 클라이언트 검토
   3.3 SEO 최적화 검토
   3.4 최종 발행

→ 각 항목에 담당자, 기한, 의존성 추가
```

## 3. 역방산 일정 수립

```
납기일에서 역으로 계산:

납기: 4/30

4/28 — 최종 납품
4/25~27 — 클라이언트 최종 검토 (3일 여유)
4/22~24 — 수정 반영 (2일)
4/15~21 — 클라이언트 1차 검토 (5영업일 SLA)
4/10~14 — 내부 QC + 수정 (3일)
4/1~9 — 전체 콘텐츠 작성 (7영업일)
3/25~29 — 전략 수립 + 브리프 작성 (3일)
3/20~24 — 현황 감사 (3일)
착수: 3/20
```

## 4. RACI 매트릭스

```
R - Responsible: 실제 작업 담당
A - Accountable: 최종 책임자 (결과에 책임)
C - Consulted: 의견 제공 (양방향 소통)
I - Informed: 결과 공유 받음 (일방 소통)

예시: 월간 리포트

| 역할 | 데이터 수집 | 리포트 작성 | 검토 | 클라이언트 발송 |
|------|----------|----------|------|-------------|
| PM | R | A | A | A |
| 분석담당 | R | R | C | I |
| 팀 리드 | I | C | R | I |
| 클라이언트 | I | I | C | I |
```

```markdown
  - 모든 콘텐츠 클라이언트 승인 완료

M4: 프로젝트 완료 — 5/5
  - 발행 완료
  - 최종 정산
  - 회고 미팅
```

## 6. 안티패턴

- **계획 없이 시작**: 범위 불명확 → 지속적 변경 요청
- **버퍼 없는 일정**: 100% 꽉 찬 계획 → 첫 지연이 전체 지연
- **의존성 미파악**: A 완료 후 B 시작인데 일정 겹침
- **단일 담당자**: 한 사람이 다 맡음 → 리스크 집중
- **계획 후 방치**: 프로젝트 진행 중 계획 업데이트 없음
