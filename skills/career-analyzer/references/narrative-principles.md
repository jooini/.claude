# narrative-principles

Analyzer가 생성하는 모든 문장에 적용되는 원칙. 위반 시 문장 다시 쓸 것.

## 1. 출처 없는 주장 금지

- 단정 문장 뒤에 `(source: portrait §N)` 또는 `(source: prev §N)` 주석
- 출처를 못 붙이는 문장 → **삭제** 또는 "관찰 없음" 명시

## 2. 부정형 해명 금지

- ❌ "장애 없이 운영했다" (장애가 있었다는 맥락 암시)
- ❌ "품질이 나쁘지 않다"
- ✅ "운영 기간 중 측정된 에러율 0.X%" (양성 사실)
- ✅ 데이터 없으면 쓰지 말 것

## 3. 모호한 형용사 금지

금지 단어 (수치 없이 단독 사용):
- "많은", "훌륭한", "상당한", "크게", "놀라운", "눈에 띄는"
- "대부분", "대체로", "주로" (정확한 비율 없으면)
- "다양한", "여러" (숫자 없으면)

대체:
- "많은 커밋" → "N건 커밋"
- "주요 기여" → "커밋 비중 X%"

## 4. 수치 과장 금지

- portrait에 있는 raw 수치보다 큰 숫자 만들지 말 것
- 평균·중앙값 재계산 시 반드시 **근거 수치 병기**
- "수십", "수백" 같은 단위 표현 금지 → "N개"

## 5. 비교 표현 OK, 가치 판단 신중

- ✅ "refactor가 fix의 2.7배"
- ⚠️ "refactor 지향적" (가치 판단 — 이유 있을 때만, 단 한 번)
- ❌ "refactor를 사랑한다" (정서 표현)

## 6. 시점 구분 명확히

- "지난 사이클 대비" 언급할 때 prev 데이터 인용 필수
- prev 없으면 "지난 대비" 문구 쓰지 말 것
- "앞으로는" 같은 미래형은 §7 Goals 섹션에만

## 7. 자기 홍보 / 자기 비판 억제

- 감정 단어 금지: "자랑스럽다", "아쉽다", "죄송하다", "부끄럽다"
- 사실 + 데이터로 독자가 판단하게 두기
- 강점/약점은 있는 대로, 포장하지 말 것

## 8. 회사/사람 언급 규칙

### internal audience
- 회사명·팀명·실명 허용
- 고객사명은 당사자에게 민감할 수 있음 — 공개 자료로 확인된 것만

### external audience
- `redaction-rules.md` 전면 적용
- 실명 → 역할, 회사명 → 도메인 (e.g., "EdTech platform")

## 9. Partial/Miss 서술

- Miss goal을 축소하거나 숨기지 말 것
- 이유를 명시 (우선순위 변화 / 측정 오류 / 일정 비현실 중 택1)
- "이번엔 실패했다. 이유: X. 교체: Y" 구조

## 10. 문장 길이

- 1문장 30단어 이내
- 길어지면 분리 (근거 / 결론 / 후속 행동)
- 번역 투 피하기 ("것이다", "수 있다" 남발 금지)

## 나쁜 예 vs 좋은 예

### 나쁜 예

> "이번 분기에는 SSO 영역에서 정말 많은 기여를 했고, 팀에서도 눈에 띄는 성과를 냈다고 볼 수 있다."

### 좋은 예

> "이번 분기 SSO 그룹(identity-hub / identity-keycloak) 커밋 비중이 42% (source: portrait §9). 직전 분기 28% 대비 14%p 증가 (source: prev §9)."
