# 영수증 분석 및 정산 문서 생성

## 목적

사용자가 제공한 폴더 경로의 영수증/결제 스크린샷 이미지들을 일괄 분석하여, 중복을 제거하고 날짜별 정산 문서(마크다운)를 생성한다.

## 트리거 조건

- 사용자가 영수증/결제 이미지가 담긴 폴더 경로를 제공할 때
- "영수증 정리", "택시비 정산", "결제 내역 정리" 등의 키워드 사용 시
- `/organize-receipts <폴더경로>` 로 직접 호출 시

## 실행 워크플로우

### Step 1: 이미지 수집 및 리사이즈

사용자에게 폴더 경로를 확인한 후:

```bash
# 지원 확장자: jpeg, jpg, png, heic
# 이미지가 2000px 초과 시 Claude 이미지 제한에 걸리므로 반드시 리사이즈
mkdir -p /tmp/receipts_resized
for f in "<폴더경로>"/*.{jpeg,jpg,png,HEIC}; do
  [ -f "$f" ] && sips -Z 1200 "$f" --out "/tmp/receipts_resized/$(basename "$f")" 2>/dev/null
done
```

- 리사이즈 대상: 최대 1200px (긴 변 기준)
- HEIC 파일은 sips가 자동 변환 처리

### Step 2: 이미지 분석 (OCR)

**병렬 읽기**: Read 도구로 한 번에 5장씩 병렬로 읽는다 (컨텍스트 효율).

각 이미지에서 추출할 정보:
- **결제 날짜** (YYYY-MM-DD 형식으로 정규화)
- **결제 시간** (HH:MM)
- **결제 금액** (원 단위)
- **결제 유형** (택시, 식비, 교통비 등)
- **출발지/도착지** (택시인 경우)
- **상호명** (일반 결제인 경우)
- **결제 수단** (카드사, 카드번호 끝자리)
- **결제 번호** (거래 고유 식별용)
- **이미지 파일명** (추적용)

### Step 3: 중복 제거

영수증은 같은 거래에 대해 여러 형태로 존재할 수 있다:
- **카카오T**: "거래확인증" + "택시 이용 상세" 쌍
- **카드사 앱**: "승인 알림" + "이용 내역" 쌍
- **동일 결제번호**, 또는 **동일 날짜+시간+금액** 조합으로 중복 판별

중복 제거 우선순위:
1. 결제번호가 동일한 건 → 1건으로 합침
2. 결제번호가 없으면 날짜+시간+금액이 모두 동일한 건 → 1건
3. 상세정보가 더 많은 이미지를 기준 데이터로 채택

### Step 4: 정산 문서 생성

**파일 경로**: `~/Workspace/weaversbrain/weaversbrain/Projects/misc/YYYY-MM/YYYY-MM-DD-{카테고리}-정산.md`

**문서 구조**:

```markdown
---
date: YYYY-MM-DD
type: expense-report
category: {카테고리}
period: {시작일} ~ {종료일}
total: {총합}원
count: {건수}건
---

# {카테고리} 정산 ({시작일} ~ {종료일})

## 월별 요약

| 월 | 건수 | 금액 |
|---|------|------|
| YYYY년 M월 | N건 | XX,XXX원 |
| **합계** | **N건** | **XXX,XXX원** |

## 상세 내역

| # | 날짜 | 요일 | 금액 | 출발/상호 | 도착/비고 | 결제시간 |
|---|------|------|------|----------|----------|----------|
| 1 | YYYY-MM-DD | 월 | XX,XXX원 | ... | ... | HH:MM |

## 결제 정보

- 결제 수단: {카드 정보}
```

### Step 5: 결과 보고

문서 저장 후 사용자에게 요약 출력:
- 총 이미지 수 / 고유 거래 건수
- 월별 소계
- 총합
- 저장 경로

## 지원 영수증 유형

| 유형 | 인식 패턴 | 추출 필드 |
|------|----------|----------|
| 카카오T 택시 | "택시 이용 상세", "카카오 T 거래확인증" | 날짜, 금액, 출발지, 도착지, 차량번호 |
| 일반 카드 결제 | "승인", "결제 금액" | 날짜, 금액, 상호명, 카드정보 |
| 네이버페이 | "네이버페이 결제" | 날짜, 금액, 상호명 |
| 카카오페이 | "카카오페이" | 날짜, 금액, 상호명 |
| 배달앱 | "배달의민족", "요기요", "쿠팡이츠" | 날짜, 금액, 메뉴, 배달비 |

## 주의사항

- 이미지는 반드시 1200px 이하로 리사이즈 후 분석 (Claude 이미지 크기 제한)
- 한 번에 5장씩 병렬 Read (너무 많으면 컨텍스트 초과)
- 금액은 반드시 쉼표 포함 형식으로 표시 (예: 22,400원)
- 요일은 날짜로부터 자동 계산
- Obsidian vault 경로가 없으면 mkdir -p로 생성
- 카테고리는 영수증 유형에서 자동 판별 (택시비, 식비, 교통비 등)
- 복수 카테고리가 섞여 있으면 "경비" 로 통합하거나 사용자에게 확인
