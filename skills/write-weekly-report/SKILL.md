# write-weekly-report

일일 업무 보고서를 합쳐 주간 업무 보고서를 작성한다.

## 사용법

```
/weekly              → 지난주 보고서 (월~금)
/weekly 2026-04-01   → 해당 날짜가 포함된 주 보고서
```

## 실행 절차

### 1단계: 날짜 범위 계산

- 인자가 없으면 **지난주 월~금**
- 인자가 있으면 해당 날짜가 속한 주의 **월~금**

```bash
# 지난주 월요일 계산
date -v-monday -v-7d +%Y-%m-%d
```

### 2단계: 일일 보고서 수집

Vault 경로: `~/Workspace/weaversbrain/weaversbrain/Daily/`

- `Daily/YYYY-MM/YYYY-MM-DD.md` 패턴으로 월~금 5일분 탐색
- 없는 날은 건너뛴다 (휴일/미작성)
- 각 파일을 Read로 읽는다

### 3단계: 내용 통합

일일 보고서들에서 아래를 **프로젝트/주제별로 묶어서** 재구성한다:

#### 주요 성과
- 일별이 아닌 **프로젝트별**로 그룹핑
- 각 프로젝트의 작업을 시간순으로 나열
- 핵심 커밋 해시, 수치, 결과 포함
- 완료된 작업과 진행 중인 작업 구분

#### 이슈 현황
- 해결/미해결 분류
- 중복 이슈는 합쳐서 최종 상태만 표시
- 주 초에 발생 → 주 중에 해결된 것은 [해결]로 통합

#### 다음 주 할 일
- **금요일 보고서의 "내일 할 일"**을 기반으로 구성
- 주 중 미완료/이월 항목 포함
- 확인 필요 사항 중 미해결 건 포함

#### 확인 필요 사항
- 주 전체에서 미해결로 남은 것만

### 4단계: 문서 생성

경로: `~/Workspace/weaversbrain/weaversbrain/Daily/YYYY-MM/YYYY-WNN-weekly.md`
(예: `2026-04/2026-W14-weekly.md`)

```markdown
---
date: "YYYY-MM-DD"
type: weekly
tags: [weekly]
week: "YYYY-WNN"
period: "MM/DD(월) ~ MM/DD(금)"
---

# 주간 업무 보고서


## 기간

YYYY년 M월 D일(월) ~ M월 D일(금)


---


## 주간 요약

- {한 줄 요약 1}
- {한 줄 요약 2}
- ...


---


## 주요 성과


### {프로젝트명}

#### 완료
- 작업 내용 (커밋/수치)
- ...

#### 진행 중
- 작업 내용 (현재 상태)
- ...


---


## 이슈 현황


### [해결] {이슈}
- 원인 / 해결

### [미해결] {이슈}
- 현상 / 필요 조치


---


## 다음 주 할 일

### {주제}
- [ ] 할 일 1
- [ ] 할 일 2


---


## 확인 필요 사항

- 내용
- ...


---


## 일일 보고서

- [[Daily/YYYY-MM/YYYY-MM-DD]]
- ...
```

### 5단계: 주간 요약 작성

보고서 최상단 "주간 요약"에 3~5줄로 핵심 성과를 요약한다:
- 가장 영향력 큰 작업 우선
- 정량 수치 포함 (커밋 수, 해결 이슈 수 등)

### 6단계: 저장 확인

1. 기존 파일이 있으면 덮어쓸지 확인
2. 저장 경로 + 요약 출력 (프로젝트 수, 성과 항목 수, 미해결 이슈 수)
3. obsidian:// URI 출력

## 주의사항

- 일일 보고서에 있는 내용만 사용. 추측/추가 금지.
- 중복 제거: 같은 작업이 여러 날에 걸쳐 있으면 통합.
- 프로젝트 순서: 작업량 많은 순.
- 민감 정보 마스킹.
