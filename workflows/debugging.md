# 디버깅 규칙

에러/버그 발생 시 반드시 순서. 추측 금지.

## 7단계 절차

1. **재현** — 에러 재현 최소 명령/시나리오
2. **수집** — 로그, 스택트레이스, 변수 상태, DB 데이터
3. **범위 축소** — 네트워크/DB/로직/설정 중 어디
4. **가설** — 사실 기반 1~2개 도출
5. **검증** — 로그 추가, 디버그 출력, 조건 변경
6. **수정** — 검증된 원인만
7. **확인** — 재현 안 되는지 검증

## 금지 사항

- "이거 아닐까?" 추측 수정 금지
- 2회 실패 시 접근 방식 재검토
- 3회 실패 시 `codex:rescue` foreground

## 관련 도구

- `/debug` 스킬 — 자동 진단 (재현→로그→가설→수정)
- `Skill(debug-master)` 에이전트 — 체계적 디버깅
